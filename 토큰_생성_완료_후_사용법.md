## 🆘 문제 해결

### "remote origin already exists" 오류

이 오류는 이미 원격 저장소가 연결되어 있다는 의미입니다. 정상입니다!

확인하려면:
```powershell
git remote -v
```

결과 예시:
```
origin  https://github.com/goalsgo1/ftp-web-app.git (fetch)
origin  https://github.com/goalsgo1/ftp-web-app.git (push)
```

### "Authentication failed" 오류

**해결 방법:**
1. 토큰이 올바르게 복사되었는지 확인
2. 토큰 전체를 붙여넣었는지 확인 (공백 없이)
3. Username이 `goalsgo1`인지 확인
4. 토큰에 `repo` 권한이 있는지 확인

### "Permission denied" 오류

**해결 방법:**
1. 저장소 이름 확인: `ftp-web-app`
2. 저장소 접근 권한 확인
3. 토큰 권한 확인 (`repo` 체크되었는지)

## 🎯 다음 단계 (푸시 완료 후)

### 1. GitHub Actions 자동 배포 설정

`.github/workflows/deploy.yml` 파일이 이미 있으므로, GitHub Secrets만 설정하면 됩니다:

1. **저장소 Settings로 이동**
   - https://github.com/goalsgo1/ftp-web-app/settings

2. **Secrets and variables** → **Actions** 클릭

3. **New repository secret** 클릭하여 다음 추가:
   - `DEPLOY_HOST`: 서버 IP 주소
   - `DEPLOY_USER`: 배포 사용자명 (예: `deploy`)
   - `DEPLOY_KEY`: SSH 개인키

자세한 내용: `단계별_배포_가이드.md`의 "7. GitHub Actions CI/CD 설정" 섹션 참고
