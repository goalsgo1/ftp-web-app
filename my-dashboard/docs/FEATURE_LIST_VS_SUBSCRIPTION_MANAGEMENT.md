# 웹 기능 목록 vs 구독 관리 화면 차별화 가이드

> **작성일**: 2025-01-XX  
> **목적**: 두 화면의 차별점을 명확히 하고, 각 화면의 고유 기능을 정의

---

## 📊 현재 상황 분석

### 공통점 (문제점)
- 두 화면이 거의 동일한 레이아웃과 기능을 제공
- 사용자가 어떤 화면에서 무엇을 해야 할지 혼란스러움
- 통계 카드가 유사함
- 카드 디자인과 기능이 거의 동일

---

## 🎯 화면별 목적 및 차별점

### 1. **웹 기능 목록** (FeatureList)
**목적**: **탐색과 발견 (Discovery)**

#### 핵심 목적
- 새로운 기능을 찾고 탐색하는 곳
- 모든 사용 가능한 기능을 한눈에 볼 수 있는 곳
- 기능을 등록하고 공유하는 곳

#### 주요 기능
1. ✅ **기능 등록** (생성자만)
2. ✅ **기능 수정/삭제** (생성자만)
3. ✅ **구독하기/구독 취소**
4. ✅ **검색 및 필터링**
5. ⏳ **정렬 옵션** (추가 필요)
6. ⏳ **추천 기능 섹션** (추가 필요)
7. ⏳ **인기 기능 표시** (추가 필요)

#### 통계 카드 (현재)
- 전체 기능
- 구독 중
- 활성 알림

#### 통계 카드 (개선안)
- **전체 기능**: 등록된 모든 기능 수
- **신규 기능**: 최근 7일 내 등록된 기능 수
- **인기 기능**: 구독자가 가장 많은 기능 수

#### UI 개선안
- **탐색 중심 디자인**
  - 큰 검색창과 강력한 필터링
  - 정렬 옵션 (최신순, 인기순, 이름순)
  - 추천 기능 섹션 (상단에 별도 섹션)
  - 카테고리별 그리드 뷰
- **기능 상세 정보 강조**
  - 구독자 수 표시
  - 최근 업데이트 날짜
  - 기능 평점/리뷰 (향후)
- **액션 버튼**
  - "구독하기" 버튼이 더 눈에 띄게
  - "기능 등록" 버튼 강조

---

### 2. **구독 관리** (SubscriptionManagement)
**목적**: **관리와 모니터링 (Management)**

#### 핵심 목적
- 내가 구독한 기능들을 관리하는 곳
- 알림 설정을 변경하는 곳
- 구독 상태를 모니터링하는 곳

#### 주요 기능
1. ✅ **구독 취소**
2. ✅ **알림 활성화/비활성화** (토글)
3. ✅ **검색 및 필터링**
4. ⏳ **알림 설정 상세 관리** (추가 필요)
5. ⏳ **구독 일자 표시** (추가 필요)
6. ⏳ **최근 활동 내역** (추가 필요)
7. ⏳ **일괄 관리 기능** (추가 필요)

#### 통계 카드 (현재)
- 전체 구독
- 알림 활성화
- 알림 비활성화

#### 통계 카드 (개선안)
- **전체 구독**: 구독 중인 기능 수
- **알림 활성화**: 알림이 켜진 기능 수
- **최근 활동**: 최근 7일 내 알림 받은 횟수

#### UI 개선안
- **관리 중심 디자인**
  - 알림 설정 토글이 더 눈에 띄게
  - 각 카드에 구독 일자 표시
  - 알림 히스토리 링크
  - 일괄 선택 및 일괄 관리 기능
- **상세 정보 표시**
  - 구독 시작일
  - 마지막 알림 받은 날짜
  - 알림 수신 횟수 통계
- **액션 버튼**
  - "알림 설정" 버튼 강조
  - "구독 취소"는 보조 버튼으로
  - "알림 히스토리 보기" 링크

## 🔄 차별화 전략

### 2. 카드 디자인 차별화

#### 웹 기능 목록 카드
- **탐색 중심**
  - 큰 썸네일 이미지 (향후)
  - 구독자 수 배지
  - "신규" / "인기" 배지
  - "구독하기" 버튼이 주요 액션

#### 구독 관리 카드
- **관리 중심**
  - 알림 상태가 눈에 띄게 표시
  - 구독 일자 표시
  - 알림 설정 토글이 주요 액션
  - "구독 취소"는 보조 버튼

### 3. 기능 차별화

#### 웹 기능 목록에만 있는 기능
- ✅ 기능 등록
- ✅ 기능 수정/삭제 (생성자만)
- ⏳ 정렬 옵션 (최신순, 인기순, 이름순)
- ⏳ 추천 기능 섹션
- ⏳ 인기 기능 표시
- ⏳ 구독자 수 표시

#### 구독 관리에만 있는 기능
- ✅ 알림 활성화/비활성화 토글
- ⏳ 구독 일자 표시
- ⏳ 최근 활동 내역
- ⏳ 알림 히스토리 링크
- ⏳ 일괄 선택 및 일괄 관리
- ⏳ 알림 수신 통계

## 📋 구현 우선순위

### Phase 1: 즉시 구현 가능 (높은 우선순위)

#### 웹 기능 목록
1. **정렬 옵션 추가**
   - 최신순 (기본)
   - 인기순 (구독자 수 기준)
   - 이름순 (가나다순)

2. **구독자 수 표시**
   - 각 기능 카드에 구독자 수 배지 추가

3. **통계 카드 개선**
   - "신규 기능" 통계 추가
   - "인기 기능" 통계 추가

#### 구독 관리
1. **알림 설정 토글 강조**
   - 토글 버튼을 더 크고 눈에 띄게
   - 토글 상태를 색상으로 구분

2. **구독 일자 표시**
   - 각 카드에 "구독 시작일: YYYY-MM-DD" 표시

3. **통계 카드 개선**
   - "최근 활동" 통계 추가

### Phase 2: 중기 구현 (중간 우선순위)

#### 웹 기능 목록
1. **추천 기능 섹션**
   - 상단에 별도 섹션으로 표시
   - 최대 3개 기능 추천

2. **인기 기능 배지**
   - 구독자 수가 많은 기능에 "인기" 배지 표시

#### 구독 관리
1. **최근 활동 내역**
   - 각 카드에 "마지막 알림: YYYY-MM-DD" 표시

2. **알림 히스토리 링크**
   - 각 카드에 "알림 히스토리 보기" 링크 추가

### Phase 3: 장기 구현 (낮은 우선순위)

#### 웹 기능 목록
1. **기능 평점/리뷰 시스템**
2. **썸네일 이미지 지원**

#### 구독 관리
1. **일괄 선택 및 일괄 관리**
   - 여러 기능 선택
   - 일괄 알림 설정 변경
   - 일괄 구독 취소

2. **알림 수신 통계**
   - 각 기능별 알림 수신 횟수
   - 주간/월간 통계
