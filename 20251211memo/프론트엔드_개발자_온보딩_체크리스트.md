# 프론트엔드 개발자 온보딩 체크리스트

> OptimonWeb 프로젝트 프론트엔드 개발자를 위한 단계별 가이드

---

## 📋 Phase 1: 프로젝트 환경 파악 및 설정 (1-2일)

### 1.1 개발 환경 확인
- [ ] **Java 개발 환경 확인**
  - Java 17 설치 여부 확인
  - Maven 설치 여부 확인 (`mvn --version`)
  
- [ ] **프로젝트 빌드 및 실행**
  ```bash
  # 프로젝트 빌드
  mvn clean compile
  
  # 애플리케이션 실행 (포트: 58080)
  mvn spring-boot:run
  
  # 또는 설정 파일 지정
  mvn spring-boot:run -Dspring.config.location=myconfig/application.yml
  ```
  
- [ ] **접속 확인**
  - http://localhost:58080/optimonweb 접속
  - 로그인 페이지 확인 (admin / admin123)
  - 기본 화면 동작 확인

### 1.2 기술 스택 파악
- [ ] **프론트엔드 기술 스택 정리**
  - ✅ **템플릿 엔진**: Thymeleaf
  - ✅ **JavaScript 라이브러리**: jQuery 3.7.1
  - ✅ **UI 프레임워크**: Kendo UI
  - ✅ **CSS 프레임워크**: Bootstrap
  - ✅ **다국어**: i18next (한국어/영어)
  - ✅ **코드 에디터**: Monaco Editor
  - ✅ **기타**: JSZip (Excel Export), Font Awesome

### 1.3 프로젝트 구조 이해
- [ ] **프론트엔드 파일 위치 확인**
  ```
  src/main/resources/
  ├── static/              # 정적 리소스
  │   ├── js/             # JavaScript 파일들
  │   │   ├── common.js   # 공통 함수
  │   │   ├── app.js      # 메뉴 시스템
  │   │   └── [페이지별].js
  │   ├── css/            # 스타일시트
  │   ├── plugins/        # 외부 라이브러리
  │   └── img/            # 이미지
  └── templates/           # Thymeleaf 템플릿
      ├── layout/         # 레이아웃
      └── [페이지별].html
  ```

## 📋 Phase 2: 코드 구조 분석 (3-5일)

### 2.1 공통 코드 분석
- [ ] **`common.js` 분석**
  - 다국어 초기화 (`i18next`)
  - 쿠키 관리 함수
  - 사용자 정보 로드
  - 전역 변수 (`contextPath`, `userInfo`, `gridPageSize`)

- [ ] **`app.js` 분석**
  - 메뉴 시스템 구조
  - 사이드바 토글 기능
  - Flyout 메뉴 동작
  - 메뉴 렌더링 로직

- [ ] **레이아웃 구조 이해**
  - `layout.html`: 메인 레이아웃
  - `header.html`: 헤더 영역
  - `sidemenu.html`: 사이드 메뉴
  - `usermenu.html`: 사용자 메뉴

### 2.2 API 호출 패턴 파악
- [ ] **API 호출 방식 확인**
  - jQuery `$.ajax()` 사용
  - 기본 URL: `contextPath + "api/..."`
  - 인증 방식: 세션 기반 (Spring Security)
  
- [ ] **주요 API 엔드포인트 파악**
  ```javascript
  // 예시: 사용자 정보
  $.ajax({
    url: contextPath + "api/user/logininfo",
    method: "GET",
    async: false,
    success: function(res) { ... }
  });
  ```

- [ ] **API 응답 형식 확인**
  - 현재는 일관성 없는 응답 형식 사용
  - 일부는 `ApiResponse<T>` 사용 (향후 통일 예정)

### 2.3 Kendo UI 사용 패턴 파악
- [ ] **주요 Kendo 컴포넌트 확인**
  - Kendo Grid (데이터 그리드)
  - Kendo DropDownList (드롭다운)
  - Kendo ToolBar (툴바)
  - Kendo Chart (차트)
  - Kendo Upload (파일 업로드)

- [ ] **Kendo 설정 패턴 확인**
  - 데이터 소스 설정
  - 이벤트 핸들러
  - 로컬라이제이션 설정

### 2.4 다국어 시스템 이해
- [ ] **번역 파일 위치**
  - `static/js/locale/ko/translation.json.js`
  - `static/js/locale/en/translation.json.js`

- [ ] **다국어 사용 방법**
  ```javascript
  // JavaScript에서
  i18next.t('key.name')
  
  // HTML에서
  <span data-i18n="key.name"></span>
  ```

## ⚠️ 주의사항

### 보안 관련
- CSRF 보호가 현재 비활성화되어 있음 (향후 활성화 예정)
- API 호출 시 인증 필요

### 코드 품질
- 일부 코드에 `System.out.println` 사용 (로거로 변경 예정)
- API 응답 형식이 일관되지 않음 (향후 통일 예정)

### 성능
- 대용량 데이터 처리 시 성능 이슈 가능
- 그리드 가상 스크롤링 고려 필요
