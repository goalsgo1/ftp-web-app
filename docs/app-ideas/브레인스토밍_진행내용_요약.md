### 핵심 컨셉
**웹 기반 기능 + 푸시알림 전용 앱 통합 플랫폼**

개발자가 만든 여러 웹 기능들을 하나의 푸시알림 전용 앱으로 통합 관리하는 시스템입니다. 각 기능은 웹으로 구현되어 빠르게 개발하고 배포할 수 있으며, 푸시알림은 전용 앱을 통해 통합적으로 관리합니다.

## 💡 핵심 아이디어 (3가지)

### 1. 웹 기반 기능 + 푸시알림 전용 앱 통합 시스템
**날짜**: 2025-12-01 01:36

**문제 해결**:
- 과거 프로젝트의 문제점 해결
  - 푸시알림 실패 문제
  - 로그인 인증 문제
  - 앱스토어/플레이스토어 배포 과정의 번거로움

**해결 방법**:
- 모든 기능은 웹으로 구현
- 푸시알림만 전용 앱(안드로이드/iOS 각각 하나씩)에서 통합 관리
- 기능별, 시간대별 알림 제어 가능

**주요 기능**:
- 웹 기능 리스트 확인
- 구독 중인 기능 관리
- 기능별 푸시알림 on/off 설정
- 시간대별 알림 수신 설정 (예: 오전 9시~오후 3시 차단)

---

### 2. 웹-앱 인증 동기화 및 자동 로그인 시스템
**날짜**: 2025-12-01 01:43

**문제 해결**:
- 웹 세션 만료 시 자동 로그인 필요
- 앱의 로그인 상태를 활용한 웹 자동 로그인

**해결 방법** (하이브리드 접근):
1. **앱이 설치되어 있고 로그인된 경우**
   - Custom URL Scheme으로 앱 실행
   - 앱에서 즉시 토큰 생성 → 웹으로 전달

2. **앱이 설치되어 있지만 로그인 안 된 경우**
   - 앱 실행 → 앱 내에서 로그인 유도 → 로그인 후 토큰 전달

3. **앱이 설치되어 있지 않은 경우**
   - 백엔드 API로 앱 설치 여부 확인
   - 일반 웹 로그인 진행 또는 앱 설치 안내

**사용 기술**:
- Custom URL Scheme
- Firebase Dynamic Links
- Universal Links (iOS) / App Links (Android)
- 바이오메트릭 인증 (앱)

---

### 3. 기술 스택 최종 확정
**날짜**: 2025-12-01 01:47

#### 앱 개발: Flutter ✅
**이유**:
- 하나의 코드베이스로 iOS/Android 동시 개발
- 개발 시간 50% 절감
- 필요한 기능 대부분 플러그인으로 제공 (FCM, Firebase, 딥링크, 바이오메트릭 등)

**주요 패키지**:
- `firebase_core`, `firebase_messaging`, `firebase_auth`
- `url_launcher` (딥링크)
- `local_auth` (바이오메트릭)
- `shared_preferences` (로컬 설정)

#### 웹 개발: Next.js 14 + React + TypeScript ✅
**이유**:
- AI 기능 통합 용이 (Vercel AI SDK)
- 서버 컴포넌트로 AI 기능 안전하게 처리
- 풍부한 라이브러리 생태계
- 검증된 기술 스택

**주요 라이브러리**:
- **AI**: Vercel AI SDK (`ai` 패키지)
- **차트**: Recharts 또는 Apache ECharts
- **테이블**: TanStack Table
- **스타일**: Tailwind CSS
- **상태관리**: Zustand 또는 React Query

## 🏗️ 아키텍처 구조

### 전체 흐름
```
웹 기능 → 백엔드 서버 → 푸시 서버 (FCM/APNs) → 푸시알림 전용 앱 → 필터링 로직 → 사용자에게 알림 표시
```

### 주요 구성 요소
1. **웹 기능들**: Next.js로 구현, 각 기능은 독립적으로 개발/배포
2. **백엔드**: Next.js API Routes 또는 Firebase Functions
3. **데이터베이스**: Firebase Firestore
4. **인증**: Firebase Authentication
5. **푸시알림**: FCM (Android) / APNs (iOS)
6. **전용 앱**: Flutter로 개발, 푸시알림 통합 관리

### Q1: 웹 기반 기능 + 푸시알림 전용 앱 시스템이 가능할까?
