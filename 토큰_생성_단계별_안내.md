## 📋 토큰 생성 상세 단계

### Step 1: 토큰 이름 설정
- **Note**: `FTP-Web-App-Deploy` (또는 원하는 이름)
- 저장소 이름을 포함하면 나중에 관리하기 쉬움

### Step 2: 만료 기간 설정
- **Expiration**: 
  - `90 days` (권장)
  - 또는 `No expiration` (만료 없음 - 보안상 비추천)

### Step 3: 권한(Scopes) 선택
- ✅ **`repo`** 체크박스 선택 (가장 중요!)
  - 체크하면 하위 항목들이 자동 선택됨
  - 다음 항목들이 포함됩니다:
    - `repo:status`
    - `repo_deployment`
    - `public_repo`
    - `repo:invite`
    - `security_events`
    - `workflow` (GitHub Actions 사용 시)

### Step 4: 토큰 생성
- **"Generate token"** 버튼 클릭 (맨 아래)
- 페이지 맨 아래로 스크롤 필요할 수 있음

### Step 5: 토큰 복사 (⚠️ 중요!)
- 생성된 토큰이 나타남
- `ghp_`로 시작하는 긴 문자열
- 예: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
- **즉시 복사**하세요!
- 안전한 곳에 저장 (비밀번호 관리자 권장)

⚠️ **경고**: 이 페이지를 떠나면 다시 볼 수 없습니다!

## 🚀 토큰 생성 후 사용 방법

### 1. 원격 저장소 연결 (아직 안 했다면)

PowerShell에서:

```powershell

git remote -v
```

### 2. 코드 푸시

```powershell

```

**인증 정보 입력:**
- **Username**: `goalsgo1`
- **Password**: 방금 생성한 토큰 (전체 복사해서 붙여넣기)

## 💡 팁

토큰은 다음 경로에서 관리할 수 있습니다:
- Settings → Developer settings → Personal access tokens → Tokens (classic)

## 🆘 문제 해결

### 토큰을 잃어버렸다면?
1. 기존 토큰 삭제 (Settings에서)
2. 새 토큰 생성
3. `git push` 다시 시도

### 푸시 시 "Authentication failed" 오류?
- Username과 Password(토큰) 확인
- 토큰이 `ghp_`로 시작하는지 확인
- 토큰 전체를 복사했는지 확인 (공백 없이)

### 권한 오류?
- `repo` 권한이 체크되어 있는지 확인
- 토큰을 다시 생성하고 `repo` 권한 확인
